# AgenticX-AiSMS：基于多智能体的全链路通信安全与营销服务平台

## 项目背景与定位

AgenticX-AiSMS 是一个基于企业级多智能体框架（AgenticX）和开源个人 AI 助手框架（OpenClaw）构建的综合性通信平台。

项目的初始定位是**"AI + 短信安全分析"**，旨在利用大语言模型（LLM）对短信内容进行深度语义分析、意图识别和风险拦截。随着调研的深入，特别是结合了业界最新的**"5G 消息 + OpenClaw"**融合方案以及**"新通话"**技术，项目的定位已扩展为**"安全防御 + 主动营销 + 智能服务"**的三轮驱动平台。

核心理念是将**短信中台、5G 消息、新通话**封装成统一的 AI 可调用技能（OpenClawd Skill），让 AI 智能体能直接调用三大通信能力，形成自动化触达闭环。

---

## 核心技术架构：三通道统一与 AI 编排

![三通道统一技术架构图](research/images/architecture_three_channel.png)

本平台的技术架构分为三层：

### 1. 触达通道层（三通道联动）
*   **传统短信**：作为高优先级、广覆盖的兜底通道，用于强提醒、短链接预热和二次确认。
*   **5G 消息（RCS）**：作为富媒体交互的核心通道，支持卡片、按钮、网页内嵌，实现"零门槛"的类 App 体验。
*   **新通话（VoLTE/VoNR）**：作为高价值、高风险场景的实时交互通道，支持 AI 语音核验、实时字幕、意图识别和多模态交互 [1]。

### 2. 智能交互层（OpenClaw 前端 Agent）
*   **统一 Skill 入口**：业务方或 AI 只需调用统一接口（如 `clawd call msg {手机号, 内容, 类型}`），OpenClaw 自动根据用户画像、终端能力和内容敏感度选择最佳通道（短信/5G消息/新通话），并支持失败自动降级。
*   **用户意图理解**：在 5G 消息对话或新通话中，实时理解用户意图（如查询、办理、投诉），并触发相应的业务流程。

### 3. 编排与风控层（AgenticX 后端引擎）
*   **安全与策略引擎**：复用 AgenticX 的 `safety` 模块，对下发的营销内容和用户的上行回复进行双向安全检测，防范注入攻击和敏感信息泄漏。
*   **RAG 知识库检索**：结合历史诈骗案例库和企业业务知识库，为前端 Agent 提供准确的上下文支撑。
*   **动态用户画像与 CRM 闭环**：分析三通道的数据回流（送达、点击、接通、回复内容），动态更新用户画像，并自动同步至企业 CRM 系统。

## 六大核心应用场景

![六大核心应用场景图](research/images/six_core_scenarios.png)

基于上述架构，AgenticX-AiSMS 可实现以下六大自动化场景：

### 一、智能通知与告警自动化
1.  **系统异常多通道必达**：监控发现故障后，自动触发短信（高优）、5G 消息（带日志链接卡片）和新通话（外呼负责人），形成三级告警。
2.  **业务阈值触发智能提醒**：如订单超时、库存不足。根据用户等级自动分发：普通用户发短信，会员发 5G 消息卡片，VIP 用户新通话+5G消息双触达。
3.  **定时/周期性消息下发**：账单、续费提醒。提前 N 天发短信，当天发 5G 消息卡片，逾期则触发新通话。

### 二、智能客服与交互闭环
1.  **5G 消息对话流转**：用户在 5G 消息提问，简单问题直接回复，复杂问题自动发起新通话回呼，办理类业务推送 5G 消息表单。
2.  **新通话后 AI 自动跟进**：通话结束后，自动下发 5G 消息（通话摘要、满意度卡片）或短信（凭证链接），并记录工单。
3.  **用户来电智能分流**：用户呼入新通话，AI 识别意图后，查账单则挂机推 5G 消息卡片，要办理则转人工并同步 5G 消息业务页。

### 三、营销与转化自动化
1.  **智能线索培育**：筛选高潜用户，先用短信短链接预热，再发 5G 消息商品卡片，若未点击则 24 小时后新通话外呼，全链路 AI 调度。
2.  **活动/权益自动发放**：自动生成 5G 消息富媒体卡片发放优惠券，同步发短信强提醒，高价值用户新通话权益讲解。
3.  **流失/沉默用户召回**：短信发利益点短文案，5G 消息发个性化召回卡片，高价值用户新通话 AI 回访。

### 四、身份核验与安全场景
1.  **三重可信核验（金融/政务）**：高风险操作触发新通话实名语音核验，推送 5G 消息风险告知卡片，最后短信验证码二次确认。
2.  **异常行为智能阻断**：异地消费或大额交易，低风险发短信提醒，中风险发 5G 消息卡片确认，高风险立即新通话核验并提供冻结指引。

### 五、业务流程自动化（RPA + 消息）
1.  **表单/审批自动推进**：待审批事项，发短信提醒，发 5G 消息审批卡片（一键同意），超时未处理则新通话提醒。
2.  **订单/物流全链路通知**：关键节点发短信+5G消息，派件前新通话确认上门时间，签收后发 5G 消息评价卡片。
3.  **智能回访与调研**：服务后先发 5G 消息问卷，未答补发短信链接，高价值客户新通话人工回访。

### 六、统一 Skill 入口与管控
1.  **三通道统一调用**：屏蔽底层通道差异，AI 智能决策发什么、发哪路、什么时候发。
2.  **统一数据回流**：送达、阅读、点击、接通等数据统一回传给 OpenClaw 分析，自动优化后续触达策略。

## 核心技术要点与原理解析

为了支撑上述六大场景，AgenticX-AiSMS 在底层实现了多项关键技术突破，涵盖前端交互、后端编排以及核心算法。

### 1. 多通道智能路由决策引擎 (前端交互层)

![多通道智能路由决策引擎](research/images/tech_channel_routing.png)

**原理解析**：
传统的通信下发通常是单通道、静态规则的。本平台引入了基于 OpenClaw 的智能路由决策引擎。当 AI 或业务系统发起下发请求时，引擎会综合评估四个维度：**用户画像等级**、**终端支持能力**（是否支持 RCS/VoLTE）、**内容敏感度**、**时段与优先级**。
*   **动态降级机制**：若首选通道（如新通话）未接通，系统会自动降级到 5G 消息或传统短信，确保信息必达。
*   **数据回流闭环**：各通道的送达率、阅读率、点击率、接通率等数据会实时回传，用于动态更新用户画像，并反哺路由策略实现自优化。

### 2. LLM 短信安全检测双路并行架构 (核心算法层)

![LLM 短信安全检测双路并行架构](research/images/tech_llm_safety_detection.png)

**原理解析**：
在处理海量短信和 5G 消息时，单纯依赖 LLM 会导致高延迟和高成本，而单纯依赖规则又容易被绕过。本平台采用**双路并行架构 (Ensemble Architecture)** [4]：
*   **路径一（规则引擎）**：利用正则表达式、黑名单和传统机器学习（如 TF-IDF + SVM）进行毫秒级初筛。
*   **路径二（LLM 语义分析）**：利用大语言模型进行深度意图识别（Intent Classification）和思维链推理（Chain-of-Thought），准确率高达 95% 以上。
*   **策略融合层**：若规则引擎判定为高风险，则直接拦截；否则以 LLM 的深度语义判定为主。这种架构在保证拦截准确率的同时，大幅降低了系统的平均响应时间。

### 3. RAG 检索增强生成流程 (后端服务层)

![RAG 检索增强生成流程](research/images/tech_rag_pipeline.png)

**原理解析**：
为了解决 LLM 在特定垂直领域（如最新诈骗手法、企业内部业务规则）的"幻觉"问题，平台深度集成了 RAG（Retrieval-Augmented Generation）技术 [5]。
*   **向量化与检索**：将用户的上行短信或交互内容通过 Embedding 模型（如 text-embedding-3）向量化，并在向量数据库（如 FAISS/Milvus）中进行 Top-K 相似度匹配。
*   **上下文注入**：从"涉诈案例库"和"业务规则库"中提取相关上下文，与原始请求拼接成完整的 Prompt。
*   **持续学习闭环**：LLM 的判定结果和人工复审数据会定期更新回知识库，使系统的防御和交互能力随时间不断进化。

## 落地实施建议

1.  **安全基座建设**：优先复用 AgenticX 的安全组件和 RAG 机制，确保所有外发内容和用户交互的安全性。
2.  **通道打通与封装**：与彩讯等短信/5G消息平台对接，将发送能力封装为 OpenClaw 的标准 Skill。同时，探索运营商新通话 API 的接入。
3.  **场景灰度验证**：建议从"智能通知与告警"或"流失用户召回"等单一场景切入，验证三通道联动和动态用户画像的实际效果，再逐步推广至全业务流程。

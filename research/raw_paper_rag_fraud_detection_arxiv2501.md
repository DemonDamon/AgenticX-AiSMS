# Advanced Real-Time Fraud Detection Using RAG-Based LLMs

**来源**: arXiv:2501.15290v1 [cs.CR]  
**发布时间**: 2025年1月25日  
**作者**: Gurjot Singh, Prabhjot Singh (University of Waterloo, Canada), Maninder Singh (Thapar Institute of Engineering and Technology, India)  
**链接**: https://arxiv.org/abs/2501.15290

## 摘要

本文提出了一种基于 RAG（检索增强生成）技术的实时欺诈检测机制，主要针对欺诈性电话和用户冒充攻击。系统在两个维度上创新：

1. **持续更新的策略合规检查**：实时转录通话内容，使用 RAG 模型验证来电者是否在索取私人信息，确保透明度和真实性。
2. **实时用户身份验证**：通过两步验证流程确认来电者身份，确保问责制。

**核心创新**：能够在不重新训练整个模型的情况下更新策略，大幅增强系统适应性。

**实验结果**：使用合成通话录音验证，准确率达 **97.98%**，F1 分数达 **97.44%**，优于现有最先进方法。

## 核心技术架构

### RAG 在欺诈检测中的工作流程

```
公司政策文档 → 向量化嵌入 → 向量数据库存储
                                    ↓
实时通话/短信 → 特征提取(LLM) → 相似度检索 → 上下文增强 → LLM 判断 → 欺诈/正常
```

### 关键组件

**1. 公司政策文档库**
- 收集各公司/银行的员工政策和指南文档
- 这些文档提供特定场景的解释上下文
- 用于检测违反公司标准的潜在欺诈行为

**2. 嵌入与向量数据库**
- 使用嵌入模型将政策文档和提取的公司名称表示为向量
- 向量存储在数据库中，避免每次查询时重新计算
- 确保系统高效可扩展，能处理大量数据和查询

**3. LLM 特征提取**
- 从通话/短信中提取关键特征（公司名称、请求类型、敏感信息类别等）
- 将提取的特征与检索到的政策文档结合

**4. 策略合规检查**
- 将 LLM 提取的特征与 RAG 检索的相关政策进行比对
- 判断当前通话/短信是否违反合规政策

## 与 BERT 的对比优势

| 维度 | RAG-LLM 方案 | BERT 方案 |
|---|---|---|
| 策略更新 | 无需重新训练，直接更新文档库 | 需要重新标注数据并重新训练 |
| 可解释性 | 可以引用具体政策条款解释判断 | 黑盒，难以解释 |
| 泛化能力 | 通过检索新文档即可处理新场景 | 需要新场景的训练数据 |
| 准确率 | 97.98% | 低于 RAG 方案 |

## 对 AgenticX-AiSMS 的启示

这篇论文为 AgenticX-AiSMS 提供了非常直接的技术参考：

1. **短信知识库构建**：可以将已知的诈骗短信话术、欺诈模式、运营商合规政策构建为向量知识库，作为 RAG 的检索源。

2. **无需重新训练的动态更新**：当出现新型诈骗手法时，只需将新的案例和分析报告加入知识库，系统即可自动适应，无需重新训练模型。

3. **可解释性**：每条被判定为诈骗的短信，系统可以引用知识库中的相似案例或政策条款来解释判断依据，便于用户申诉和人工复核。

4. **与 AgenticX 的结合点**：AgenticX 的 Retrieval 模块（包含 GraphRAG）可以直接支持这一架构，俊雄负责的知识库安全部分可以作为 RAG 的知识源接入。

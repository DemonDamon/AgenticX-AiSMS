## 核心问题

电信网络欺诈持续演变，其文本表达越来越隐蔽，使自动化检测更具挑战性。主流提示策略（零样本、少样本、思维链）与 LLM 结合时，在处理多样化欺诈文本时往往表现不稳定，尤其是对于**长尾类别**和**混淆案例**。

## MRML 提示策略

### 三个专家角色

MRML 策略模拟人类专家小组，构建三个专家角色进行协同推理：

| 专家角色 | 职责 | 分析维度 |
|---|---|---|
| **文本分析专家** | 从语言学角度分析文本特征 | 词汇选择、句式结构、情感倾向、紧迫感制造等 |
| **业务流程分析专家** | 从业务逻辑角度分析合规性 | 请求是否符合正常业务流程、是否索取不应索取的信息 |
| **安全分析专家** | 从安全威胁角度分析风险 | 已知欺诈模式匹配、社会工程学手法识别 |

### 条件分层推理机制

MRML 采用**条件触发**的分层推理：

```
阶段1：快速二分筛查（Binary Screening）
  ↓ 若判断为"可疑"
阶段2：深度多类别分类（Multi-class Classification）
  - 三个专家角色并行分析
  - 综合判断欺诈类型
```

这种设计在不牺牲性能的前提下提高了计算效率——只有初步筛查认为可疑的内容才会触发深度分析。

## 实验结果

在两个公开数据集上的实验表明，MRML 框架：
- 显著优于主流提示策略（零样本、少样本、思维链）
- 在精确率、召回率和 F1 分数上超越深度学习基线（BERT、TextCNN、Transformer）
- 展现出更强的鲁棒性

## 技术演进路线

论文梳理了电信欺诈检测的技术演进：

**第一代：传统机器学习**
- TF-IDF + SVM/NB/RF
- SVM 在钓鱼邮件检测中达到 97.3% 准确率
- 缺点：依赖特征工程，无法适应新型欺诈

**第二代：深度学习**
- BERT、TextCNN、Transformer
- 自动特征学习，更强的语义理解
- 缺点：黑盒，可解释性差；新型欺诈缺乏标注数据

**第三代：LLM + 提示工程（当前最优）**
- 零样本/少样本泛化能力强
- 思维链推理提供可解释性
- MRML 进一步提升稳定性和多类别识别能力

## 对 AgenticX-AiSMS 的启示

1. **多角色协同是关键**：单一 LLM 对欺诈文本的判断不稳定，多角色协同分析可以显著提升稳定性。AgenticX 的多 Agent 编排能力天然支持这种多角色架构。

2. **两阶段检测提升效率**：先快速筛查，再深度分析，这种设计对于运营商级别的大规模短信处理（每天数十亿条）尤为重要。

3. **长尾类别处理**：诈骗短信的类型分布极不均衡（常见类型多，罕见类型少），MRML 的多角色设计对长尾类别有更好的处理能力。

4. **可解释性**：多角色分析自然产生可解释的推理链，每个角色的分析结果都可以作为判断依据呈现给用户。

5. **与 AgenticX 的直接映射**：
   - 文本分析专家 → AgenticX 的 NLP 分析 Agent
   - 业务流程分析专家 → AgenticX 的规则引擎 Agent（结合运营商合规知识库）
   - 安全分析专家 → AgenticX 的安全检测 Agent（结合 Llama Guard 等安全分类器）
